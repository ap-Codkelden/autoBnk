3. Описание файла tax.yaml

Представляет собой файл формата YAML с корневым элементом tax со множеством дочерних элементов row, которые в свою очередь содержат дочерние элементы query с параметрами запроса для выборки.

3.1. Элемент row

Каждый элемент row с кодом _code: 'NNNNNN' соответствует строке выходной таблицы с кодом NNNNNN, где NNNNNN соответствует коду строки из файла etalon.yaml. 

- row:
    - query:
        rozd: 1 # числовой
        bd: 0 # числовой. Обязательный
        rd: '1102' # строковый. Обязательный
        pg: '01' # строковый
    - query:
        rozd: 1
        bd: 0
        rd: '1102'
        pg: '40' 
        st: '01'
        coef: 0.90
    _code: '110200' # строковый. Обязательный

Элемент row может содержать неограниченное количество элементов query. Для каждой строки, определенной в row результат равен сумме результатов дочерних элементов query.

3.2. Элемент query

Каждый элемент query должен содержать следующие обязательные элементы:

  • bd – вид бюджета: 0 – государственный, 1 – местный;
  • rd – раздел бюджетной классификации, 4 цифры;

А также необязательные элементы (если параметр не участвует в выборке, такие элементы не указываются):

  • rozd – столбец Раздел из файла-отчёта казначейства.
  • pg – параграф бюджетной классификации, 2 цифры.
  • coef – Коэффициент. Представляет собой рациональное число с двумя или более разрядами, причём в качестве разделителя выступает точка (1.00, 0.334, -2.35). Может быть отрицательным, если необходимо обеспечить вычитание какой-либо суммы из общей суммы по строке.

    По умолчанию равен 1.00
