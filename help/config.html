4. Описание файла config.yaml

Представляет собой файл формата YAML с корневым элементом config, содержащим дочерние элементы path и treasury_files.

4.1. Элемент path

В дочерних элементах path-bank и path-out содержит информацию о каталогах:

  • path-bank — в который пользователем помещаются казначейские файлы;
  • path-out — в который сохраняется выходной HTML-файл;

Следует помнить, что специальные символы внутри двойных кавычек должны быть экранированы, поэтому при использовании двойных кавычек " бэкслэш \ в значениях параметров path-bank и path-out должен быть экранирован.

При использовании одинарных кавычек ' экранирование не нужно.

path:
  path-bank: "d:\\Autobnk\\in"
  path-out: 'd:\Autobnk\out'

4.2. Элемент treasury_files

Во множестве дочерних элементов file содержит информацию об органах государственной казначейской службы, файлы которых будут обрабатываться.

Каждый элемент file содержит следующие обязательные дочерние элементы:

  • code – код органа государствееной казначейской службы;
  • raj – код района инспекции.

В элементе file может содержаться также необязательный дочерний элемент inverse, принимающий значение true.

Наличие данного элемента необходимо для кодов казначейств, которым необходимо условие выборки, согласно которго бюджет имеет признак «сводный» - 0.

Если применение такого условия выборки не нужно, элемент inverse не указывается.

- file:
  # УДКСУ у Криворізькому районі
  code: "097"
  raj: 18
  inverse: true
